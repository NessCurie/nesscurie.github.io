将Java的byte数组分段解析为所需数据的工具类BytesReader

Jul 28, 2018 | Java
Comment

要点

  • 跨设备进行传输时一般使用byte数组,需要将byte数据解析为各种java的数据,一般为基本数据类型和字符串以及byte表示的是无符号的情况,无符号的除了无符号64位以外都可以用java基本数据类型进行接收
  • 进行字符串传输时,一般会用一段字节表示后续字符串的byte长度,比如第一字节为无符号8位表示后续的内容长度,长度后续紧跟内容
  • 有时候会将剩余数据全部获取的情况
  • 在调试时需要将byte数组使用16进制打印出来
  • 有时候会用全ff的内容表示无效数据
  • 有时候某段数据无用需要进行索引跳转
  • 跨设备转化为字符串时一般为GBK编码,有时候有额外情况
